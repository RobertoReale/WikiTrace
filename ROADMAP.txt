 2. FEATURE FUTURE
════════════════════════════════════════

FEATURE A — READING LIST ("Da leggere")
-----------------------------------------
Obiettivo: salvare pagine da leggere in futuro con categorie libere definite
dall'utente (superiore ai preferiti del browser, tutto nel contesto WikiTrace).

Funzionalità:
  - Pulsante "Salva per dopo" nel popup (separato da "Salva visitato")
  - Al salvataggio: scegli/crea una categoria utente (es. "Storia", "Da approfondire")
  - Nuova tab "Da leggere" nella dashboard con:
      • Lista filtrabile per categoria
      • Badge "non letto" / "già visitato"
      • Azione "Segna come letto" → sposta nella cronologia normale
      • Azione "Rimuovi" senza tracciare
  - Esportazione/importazione JSON include la reading list

Approccio tecnico (no backend, tutto locale):
  - Nuovo store: chrome.storage.local → readingList: [{ url, title, category, savedAt }]
  - Nuova tab in dashboard.html / dashboard.js
  - Nessuna modifica a manifest.json necessaria


FEATURE B — ACCOUNT SYNC (Google Sign-In + Firebase)
------------------------------------------------------
Obiettivo: i dati sopravvivono alla reinstallazione e si sincronizzano tra
dispositivi (incluso Firefox per Android).
Stack: Firebase Authentication + Firestore (free tier, nessun server proprio).

Passo 1 — Setup manuale (da fare tu)
  B1. Crea un progetto Firebase su https://console.firebase.google.com
  B2. Abilita Google Sign-In: Authentication → Sign-in method → Google
  B3. Crea un database Firestore (production mode, regione eur3)
  B4. Registra l'estensione come Web App → copia firebaseConfig
      Aggiungi l'URI di redirect OAuth in Google Cloud Console:
        https://<extension-id>.chromiumapp.org/
  B5. Annota: firebaseConfig e OAuth Client ID

Passo 2 — Modifiche al codice (una sessione)
  B6. Aggiungi Firebase SDK (Auth + Firestore) come file JS locali
  B7. background.js: sync al login, ad ogni salvataggio e al primo avvio
  B8. popup.html/js: pulsante "Sign in with Google", avatar utente, sign out
  B9. manifest.json: permesso "identity" + domini Firebase in host_permissions
  B10. Regole Firestore: ogni utente legge/scrive solo i propri dati
  B11. Aggiorna la Privacy Policy per menzionare Firebase/Google Cloud
       (obbligatorio per il Chrome Web Store quando i dati lasciano il browser)


FEATURE C — FIREFOX & FIREFOX FOR ANDROID (sessione separata)
--------------------------------------------------------------
  C1. Aggiungi webextension-polyfill (shim browser.* API)
  C2. Crea un secondo manifest per Firefox MV2 + script di build per due zip separati
  C3. Sostituisci chrome.identity con browser.identity per il login Google
  C4. Pubblica su AMO (addons.mozilla.org) — review spesso < 24 ore
  C5. Verifica su Firefox for Android: login, salvataggio e sync con Chrome
